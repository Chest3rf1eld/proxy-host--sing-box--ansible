# sing-box Client Ansible

Этот репозиторий автоматически подготавливает удалённый `sing-box` клиент.

Если простыми словами: он настраивает сервер, через который можно направлять трафик во внешний peer, чтобы получать доступ к сервисам, которые недоступны из определённой страны или региона.

Этот репозиторий полезен:

- инженерам, которым нужна воспроизводимая настройка удалённого `sing-box` клиента
- командам, которым важно хранить инфраструктурную настройку в коде, а не делать её вручную
- нетехническим участникам процесса, которым нужно простое объяснение назначения репозитория

## Что Делает Этот Репозиторий

После запуска playbook целевой сервер подготавливается для работы как `sing-box` клиент:

- устанавливается пакет `sing-box`
- клиентская конфигурация собирается из переменных
- конфигурация проверяется перед выкладкой
- сервис `sing-box` включается и запускается

## Типовой Сценарий

Основной сценарий использования: этот хост работает как исходящий клиент, через который можно подключаться к внешним сервисам, заблокированным или недоступным из текущей страны.

Конфигурация из примера также включает TUN inbound. Он перехватывает исходящий трафик самого сервера, проксирует только домены ChatGPT/OpenAI/Codex CLI через outbound `proxy`, а весь остальной трафик отправляет напрямую через outbound `direct`.

Для Codex CLI это покрывает оба обычных сценария: ChatGPT-login через `chatgpt.com` и API-key режим через `api.openai.com`, потому что `api.openai.com` попадает под правило `openai.com`.

Локальный mixed proxy на `127.0.0.1:2080` остаётся доступен, но теперь следует тем же правилам маршрутизации, а не отправляет весь трафик в прокси принудительно.

## Структура

- `site.yml` — основной playbook
- `inventory/hosts.yml.example` — пример inventory с форматом целевого хоста
- `group_vars/sing_box.yml.example` — пример клиентской конфигурации `sing-box`
- `roles/sing_box/` — Ansible-роль, которая выполняет установку и настройку

## Локальные файлы

Эти файлы должны оставаться только на локальной машине и игнорируются Git:

- `inventory/hosts.yml`
- `group_vars/sing_box.yml`

Создай их на основе файлов-примеров и замени тестовые значения на реальные.

## Использование

1. Создай локальные файлы из шаблонов:
   - `cp inventory/hosts.yml.example inventory/hosts.yml`
   - `cp group_vars/sing_box.yml.example group_vars/sing_box.yml`
2. Заполни локальные файлы реальными параметрами хоста и конфигурацией `sing-box`.
3. Запусти playbook:

```bash
ansible-playbook site.yml
```

После применения можно проверить Codex CLI на целевом сервере:

```bash
codex --version
codex exec "ping"
```

Если нужно быстро проверить именно локальный mixed proxy без TUN-перехвата:

```bash
HTTPS_PROXY=http://127.0.0.1:2080 codex exec "ping"
```

## Что Делает Роль

Роль `sing_box`:

- при необходимости настраивает официальный APT-репозиторий на Debian-based системах
- устанавливает `sing-box` из APT-пакета на Debian-based системах
- устанавливает `sing-box` из GitHub release archive на RedHat-family системах, включая Sangoma OS
- рендерит `/etc/sing-box/config.json`
- проверяет конфигурацию перед применением
- включает и запускает сервис `sing-box`

## Как Получить Конфиг

Проще всего получить рабочую конфигурацию `sing-box` через экспорт из Nekoray:

1. Открой нужный peer в Nekoray.
2. Нажми `Share`.
3. Выбери `Export sing-box config`.
4. Перенеси нужные значения в `group_vars/sing_box.yml`.

## Примечания

- Runtime-конфигурация хранится в `group_vars/sing_box.yml`, а не в defaults роли.
- Секреты, хост-специфичные значения и пути к приватным SSH-ключам должны храниться только в локальных файлах.
- TUN-режим требует прав на создание сетевого интерфейса и изменение маршрутов; штатный systemd-сервис `sing-box` обычно запускается с нужными правами.
- Если Codex CLI использует вход через Google, Microsoft или Apple, сам OpenAI-трафик будет проксироваться, но внешний OAuth-провайдер может потребовать отдельного правила.
- Для хостов без прямого доступа в интернет, где `sing-box` уже установлен, можно задать `sing_box_install_package: false`; тогда роль пропустит настройку APT-репозитория, `apt-get update` и установку пакета.
- На Sangoma OS стандартное значение `sing_box_install_method: auto` использует release archive, потому что в стандартных репозиториях нет пакета `sing-box`. Другую версию можно закрепить через `sing_box_archive_version`.
